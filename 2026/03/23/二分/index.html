算法篇之二分查找

二分查找的背景与思路

二分查找是一种在有序（或具有某种单调性质）的数据中高效定位目标的算法。

核心思想非常简单：每次取区间的中点，判断答案在中点的左侧还是右侧，从而将搜索范围缩小一半。因此，对于长度为 n 的区间，二分查找只需要 O(log n) 次操作即可找到目标。

二分的本质并不局限于”在有序数组中查找某个数”。更一般地说，只要一个性质将区间分成了两段——左半段不满足、右半段满足（或反过来），我们就可以用二分查找找到这个分界点。

二分查找做的事情，就是不断缩小范围，最终精确定位到这个分界点。

整数二分

整数二分十分易错，如果边界条件没有找对，就容易陷入死循环导致超时。

所以，我认为有必要记住如下两个模板（其实记住一个，另一个也记得很快）方便我们做题。

模板一：查找左边界（r = mid）

当 check(mid) 为 true 时，说明答案在 mid 或 mid 的左边，所以令 r = mid。

要点： 当 r = mid 时，mid 的计算不需要 +1，即 mid = (l + r) >> 1。因为向下取整保证 mid < r（当 l < r 时），所以 r = mid 一定能让区间缩小，不会死循环。

典型场景： 找满足某个条件的最小值（第一个满足条件的位置）。

模板二：查找右边界（l = mid）

当 check(mid) 为 true 时，说明答案在 mid 或 mid 的右边，所以令 l = mid。

要点： 当 l = mid 时，mid 的计算必须 +1（向上取整），即 mid = (l + r + 1) >> 1。如果不加 1，当 l + 1 == r 时，mid 会等于 l，导致 l = mid = l，区间不缩小，陷入死循环。

典型场景： 找满足某个条件的最大值（最后一个满足条件的位置）。

如何选择模板？

一句话总结：看 check(mid) 成立时更新的是 l 还是 r。

check(mid) 成立时 更新方式 mid 计算 适用场景
r = mid 不加 1 (l + r) >> 1 找满足条件的最小位置
l = mid 加 1 (l + r + 1) >> 1 找满足条件的最大位置

口诀：l = mid 要加一，r = mid 不加一。

例题：数的范围（AcWing 789）

给定一个升序数组和若干个查询，每次查询一个数 $k$，输出它在数组中出现的起始位置和终止位置（若不存在输出 -1 -1）。

分析：

  • 找起始位置用模板一：我们希望找到第一个 a[mid] >= k 的位置，当条件成立时 r = mid，不断往左逼近。
  • 找终止位置用模板二：我们希望找到最后一个 a[mid] <= k 的位置，当条件成立时 l = mid，不断往右逼近。

实数二分

相比整数二分，实数二分不存在边界问题，因为实数域是连续的，不需要考虑 +1 或 -1，也不会死循环。

基本模板

关于精度： 循环终止条件 r - l > eps，其中 eps 一般取比要求精度小两个数量级。例如题目要求保留 6 位小数（精度 $10^{-6}$），则取 eps = 1e-8。

也可以用固定迭代次数的写法代替精度判断，避免精度陷阱：

迭代 100 次，区间会缩小为原来的 $\frac{1}{2^{100}} \approx 10^{-30}$，精度绰绰有余。

例题：求数的三次方根（AcWing 790）

给定一个浮点数 $n$，求它的三次方根，结果保留 6 位小数。

分析：

  • 三次函数 $f(x) = x^3$ 是单调递增的，满足二分的条件。
  • 当 mid^3 >= n 时，说明 mid 偏大或刚好，令 r = mid。
  • 当 mid^3 < n 时，说明 mid 偏小，令 l = mid。
  • 注意 $n$ 可能为负数，所以搜索范围取 $[-10000, 10000]$（根据题目数据范围确定）。

总结

整数二分 实数二分
区间更新 l = mid + 1 或 r = mid - 1 l = mid 或 r = mid
终止条件 l < r（最终 l == r） r - l > eps 或固定迭代次数
难点 边界 +1 问题，容易死循环 精度控制
mid 计算 r = mid 时不加 1；l = mid 时加 1 直接 (l + r) / 2
时间复杂度 $O(\log n)$ $O(\log \frac{R-L}{\varepsilon})$

牢记口诀：l = mid 要加一，r = mid 不加一；实数二分没烦恼，精度多开两位就好。
